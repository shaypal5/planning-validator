# Roadmap

## Current

- Keep repository planning docs aligned with recently merged pull requests.
- Review generated documentation refresh PRs before merging them.

## Next

- Tune the planning and tracking file globs as the repository grows.
- Add reviewers to the generated PRs once ownership is clear.

## Safety Boundaries

- Only markdown files listed in the config may be updated.
- Generated PRs stay on the fixed `automation/planning-validator` branch.
- Humans remain responsible for reviewing and merging documentation updates.
